# 剧本杀评测迭代管理

## 概述

本技能定义了剧本杀作品评测迭代的标准化工作流程。核心围绕一份多 Sheet 的 Excel 总表，管理从评测数据录入、维度评分更新、改进建议跟踪到实测反馈记录的全流程。

## 适用场景

- 用户完成一轮新的评测（AI 模拟或真人实测），需要将结果录入总表
- 用户修改了剧本内容，需要记录迭代决策
- 用户要求查看/更新改进清单状态
- 用户提供玩家反馈，需要录入实测反馈表
- 用户要求生成阶段性评测报告或对比分析

## 总表结构

总表为 `.xlsx` 格式，包含 7 个 Sheet：

| Sheet | 用途 | 更新方式 |
|-------|------|----------|
| **总评概览** | 版本迭代记录 + 总评分数 + 核心竞争力/短板 | 追加版本行 + 覆盖总评 |
| **各维度评分** | 信息设计/推理体验/节奏设计/社工空间的子项得分 | 覆盖分数和说明 |
| **角色体验** | 各角色的代入感/情感弧/互动质量/综合评分 | 覆盖评分 |
| **改进清单** | 改进建议跟踪（优先级/状态/实施版本/效果验证） | 更新状态 + 追加新建议 |
| **Agent状态波动** | AI Agent 情绪热力图 + 关键转折点 | 覆盖情绪标记 |
| **实测反馈** | 真人测试记录 + 玩家反馈摘录 | 追加行 |
| **迭代决策日志** | 每次修改剧本的决策/理由/涉及角色 | 追加行 |

## 标准操作流程（SOP）

### 流程一：新版本评测数据录入

当用户说"评测完了"或提供新版本评测数据时，按以下顺序执行：

1. **确认版本号和日期** — 如用户未明确，主动询问
2. **总评概览** — 在版本迭代记录表末尾追加一行（版本/日期/修改内容/修改原因/负责人），更新总评分数
3. **各维度评分** — 覆盖更新子项分数和说明，在说明末尾标注版本号如 `(v1.1更新)`
4. **角色体验** — 覆盖更新各角色评分和一句话评价
5. **Agent状态波动** — 如有 AI 模拟数据，更新情绪热力图

### 流程二：改进建议状态更新

当用户说某些建议"已实施"或"已完成"时：

1. 在**改进清单** Sheet 中定位对应建议编号
2. 将状态从 `待实施` 改为 `已实施`
3. 填入实施版本号（如 `v1.1`）
4. 如用户提供了效果数据，填入效果验证列
5. 如有新增建议，在末尾追加行

### 流程三：实测反馈录入

当用户完成真人测试并提供反馈时：

1. 在**实测反馈** Sheet 的测试记录表追加一行（测试编号/日期/人数/时长/版本/结局/整体评分/记录人）
2. 测试编号按 `T-001`, `T-002` 递增
3. 在玩家反馈摘录表追加各玩家的具体反馈

### 流程四：迭代决策记录

当用户修改了剧本内容时：

1. 在**迭代决策日志** Sheet 追加一行（日期/决策/理由/涉及角色或线索/决策人）
2. 同步更新**总评概览**的版本迭代记录

## 技术要求

### Excel 操作规范

- 使用 `openpyxl` 库操作 `.xlsx` 文件
- 加载时使用 `load_workbook(filepath)`，不要使用 `data_only=True` 以保留公式
- 追加行时：找到目标表格的最后一个有数据的行，在其下方插入
- 覆盖更新时：定位到具体单元格直接写入新值
- 保存后原文件被覆盖，无需另存

### 样式保持

- 新追加的行应继承相邻行的样式（字体、对齐、边框、填充色）
- 交替行底色规则：偶数行浅蓝（D6EAF8），奇数行白色
- 表头行蓝色填充（2E75B6）+ 白色粗体字

### 数据校验

- 版本号格式：`v主版本.次版本`（如 v1.0, v1.1, v2.0）
- 日期格式：`YYYY-MM-DD`
- 分数范围：0-100 的整数
- 改进建议编号格式：`字母-数字`（如 R-01, T-01, S-01）

## 输出规范

每次更新完成后：

1. 保存 Excel 文件
2. 简要汇总本次更新了哪些 Sheet、改了什么
3. 提醒用户可以上传到腾讯文档做备份
