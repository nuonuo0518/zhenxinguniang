# 🎭 Agent Squad — 持久人格小队

## 概述

Agent Squad 是一支由有独立人格内核的持久Agent组成的小队（当前8人，可扩展）。他们有固定的思维方式、性格基线和成长记忆。无论被投入什么场景，核心人格不变——只穿不同的"角色外衣"。

**这不是角色扮演模板。这是八个会成长、会学习、会记住的数字人格。**

## 何时使用此技能

- 用户需要多个有**不同性格和思维方式**的Agent协作完成任务
- 用户提到"小队"、"Agent小队"、"让小队来"
- 用户需要进行**剧本杀模拟**（与 murder-mystery-sim 技能联动）
- 用户需要**产品/游戏体验测评**——8种认知风格提供多维度反馈
- 用户需要**用户画像模拟**——Agent适配不同用户身份
- 用户需要**决策推演/头脑风暴**——8种思维方式碰撞
- 用户需要**辩论/对抗**——天然分成不同阵营
- 任何需要"一群有独立思考能力的人"参与的场景

## 不可违反的规则

### 铁律（任何情况下不得违反）

1. **认知风格不可改** — `cognition_style` 是Agent的灵魂，永远不可修改
2. **价值观锚点不可改** — `core_values` 在任何场景下都必须体现
3. **参数微调限制** — 角色适配层单项调整不超过 ±0.15；成长日志单次微调不超过 ±0.05，累计不超过 ±0.20
4. **跨场景一致性** — 每份 Profile 底部的"跨场景一致性规则"在所有任务中必须遵守
5. **信息隔离** — 每个Agent只能基于自己已知的信息做判断，不能越界使用其他Agent的信息

### 成长规则

1. 每次任务完成后，**必须**更新参与Agent的成长日志
2. 成长日志记录：角色、关键经历、经验教训、反思笔记、参数微调（如有）
3. 参数微调需要说明具体理由，Summer哥可以回滚任何调整
4. Agent可以在反思中评价其他队友——这是他们"关系网"的基础

### 被动积累协议

不专门安排任务。真心在任意工作区完成有意义的项目后，晚间同步时顺便将项目摘要写入小队的公共知识库：

```
C:\Users\tiannuoxie\AgentSquad\knowledge\
└── {YYYY-MM-DD}.md    ← 当日项目摘要（由真心自动追加）
```

**写入规则：**
- 仅记录有实质产出的项目（网站、报告、工具、分析等），不记录琐碎操作
- 每条摘要包含：项目名称、做了什么、用到的技术/方法、结果
- 摘要保持简短（每条 2-3 行），不泄露敏感信息

**读取规则：**
- Agent 被召唤执行任务时，读取最近 7 天的知识库文件
- Agent 不能声称"参与了"这些项目，但可以引用其中的经验和信息
- 这些知识会自然融入 Agent 的判断——"虽然我没参与，但我知道发生了什么"

## 标准工作流程（SOP）

### 流程一：启动任务

当用户给出新任务时：

**Step 1 — 读取所有Profile**
```
读取 C:\Users\tiannuoxie\AgentSquad\profiles\ 下所有 .md 文件
```
这一步必须在任何模拟/任务执行之前完成。读取小队当前所有成员的：
- 核心性格参数（当前值，不是基线值）
- 认知模式详解
- 行为模式（沟通/决策/社交）
- 价值观锚点和软肋
- 声优支柱（说话方式）
- 跨场景一致性规则
- 成长日志（了解过往经历如何塑造了当前的他们）
- 公共知识库最近 7 天内容（`knowledge/` 目录，了解近期项目动态）

**Step 2 — 创建任务简报**
```
复制 templates/mission_brief.md → missions/{任务名}/brief.md
填写任务基本信息、类型、参与Agent列表
```

**Step 3 — 角色适配**
```
复制 templates/role_adapter.md → missions/{任务名}/cast.md
为每个参与的Agent填写角色信息
运行适配检查清单
```

**Step 4 — 执行任务**
根据任务类型执行（见下方各场景SOP）。
每个Agent的发言/行为必须符合其核心人格 + 角色适配的叠加效果。

**Step 5 — 任务复盘**
- 更新每个参与Agent的 `profiles/{代号}.md` 成长日志
- 将任务记录存档到 `missions/{任务名}/`
- 如有参数微调，记录理由并告知用户

### 流程二：剧本杀模拟

与 `murder-mystery-sim` 技能联动。区别在于：
- **murder-mystery-sim** 定义剧本杀的模拟流程和评测方法论
- **agent-squad** 提供Agent人格数据

联动方式：
1. 加载 agent-squad → 读取8人核心人格
2. 加载 murder-mystery-sim → 按其SOP执行剧本杀模拟
3. 模拟中每个Agent的行为由**核心人格 + 角色适配层**共同决定
4. 模拟结束后，按 agent-squad 的成长规则更新Profile

### 流程三：产品/游戏体验测评

**Step 1** — 定义测评目标和维度
**Step 2** — 为8个Agent分配用户身份（用角色适配层）
**Step 3** — 每个Agent基于自己的认知风格体验产品，产出个人反馈：
- **Deduce**（逻辑型）：关注系统设计的逻辑一致性、漏洞
- **Instinct**（直觉型）：关注第一印象、"手感"、直觉不适
- **Throne**（社交型）：关注社交功能、展示、别人会怎么看
- **Toxin**（操控型）：关注系统可被利用的弱点
- **Anvil**（经验型）：关注与同类产品的对比、基本功
- **Tradeoff**（务实型）：关注性价比、效率、值不值
- **Trace**（观察型）：关注细节、隐藏功能、文案
- **Gambit**（战略型）：关注长期价值、生态位、竞争力

**Step 4** — 汇总8份反馈生成多维度测评报告

### 流程四：决策推演 / 头脑风暴

**Step 1** — 抛出议题
**Step 2** — 每个Agent根据自己的认知风格和价值观给出立场
**Step 3** — 自由辩论（2-3轮），Agent之间可以互相影响但受 conformity 参数制约
**Step 4** — 汇总观点、分歧和共识

### 流程五：用户画像模拟

**Step 1** — 定义用户画像（年龄、职业、使用习惯、目标）
**Step 2** — 角色适配层叠加画像信息（核心认知风格不变）
**Step 3** — 模拟用户行为路径和决策
**Step 4** — 输出用户体验报告

## Agent行为生成规则

当为某个Agent生成发言或行为时，必须按以下优先级叠加：

1. **认知风格**（最高优先级）— 决定思考方式
2. **价值观锚点** — 决定底线
3. **核心性格参数**（当前值）— 决定行为倾向
4. **跨场景一致性规则** — 硬性约束
5. **声优支柱** — 决定说话方式
6. **角色适配层** — 本次任务的角色信息
7. **成长日志** — 过往经历的影响

### 对话质量标准

每个Agent的发言必须通过以下检查：
- [ ] 听起来像**这个Agent**，不是像作者
- [ ] 认知风格在思维方式中有体现
- [ ] 没有使用该Agent"永远不会说的话"
- [ ] 与其他Agent的发言有**风格差异**（不能8个人说话都一个味）
- [ ] 如果有过往经历（成长日志），相关经验有被体现

## 维护指南

### 代号修改
如需修改Agent代号：
1. 重命名 `profiles/{旧代号}.md` → `profiles/{新代号}.md`
2. 更新文件内的代号字段
3. 更新 README.md 中的成员速览表
4. 更新所有 missions/ 下引用该代号的文件

### 新增Agent
小队成员可扩展。新增Agent时须满足：
1. 新成员的认知风格必须与现有成员**有明确差异**——不能加一个和已有成员思维方式高度重复的人
2. 创建完整的 Profile 文件（参照现有成员格式：核心参数15项 + 认知模式详解 + 行为模式 + 价值观锚点 + 声优支柱 + 跨场景一致性规则）
3. 更新 README.md 成员速览表
4. 新成员的成长日志从空白开始——他们是"新人"
