# Agent Squad 场景SOP参考

## 场景A：剧本杀模拟

> 与 murder-mystery-sim 技能联动使用

### 准备阶段
1. 读取全部8份 Profile（或指定参与的Agent）
2. 解析剧本材料（角色手册、DM手册、线索卡等）
3. 为每个Agent分配角色，创建 cast.md
4. 初始化关系矩阵
5. **适配检查**：确认所有参数调整在范围内

### 执行阶段
按 murder-mystery-sim 的标准流程执行。关键约束：
- 每个Agent的发言风格必须体现其核心认知风格
- 凶手Agent的隐藏行为受其 deception_detection 和 persuasion_skill 参数影响
- 情绪变化受 fear_susceptibility、guilt_susceptibility、empathy_susceptibility 参数调节

### 复盘阶段
1. 更新每个Agent的成长日志
2. 评估是否有参数微调需求
3. 归档任务记录

## 场景B：产品/游戏体验测评

### 准备阶段
1. 读取全部8份 Profile
2. 确定测评目标和维度
3. 为每个Agent分配"用户身份"（如适用，用角色适配层）

### 执行阶段

每个Agent按其认知风格生成测评反馈：

**Deduce（logical）— 系统逻辑审计**
- 重点：流程设计的逻辑一致性、规则漏洞、边界情况
- 风格：列举具体问题，用"如果X，那么Y"的方式暴露设计缺陷
- 参考台词："这里的奖励机制和惩罚机制矛盾——玩家被激励去做被惩罚的事。"

**Instinct（intuitive）— 第一印象体验**
- 重点：手感、节奏、直觉不适、"说不清但就是不对"的体验
- 风格：直接、感性、用比喻描述体验
- 参考台词："打开App那一瞬间我就烦了——这个加载动画在侮辱我的时间。"

**Throne（social）— 社交与展示评估**
- 重点：社交功能、分享机制、别人看到会怎么想、面子价值
- 风格：从"别人的目光"出发评价
- 参考台词："这个成就系统——我发到朋友圈别人能看懂吗？看不懂它就没有意义。"

**Toxin（logical-manipulative）— 系统可利用性分析**
- 重点：规则漏洞、可被exploit的机制、灰色地带
- 风格：冷静列举"如果有人故意这样做"的场景
- 参考台词："这个邀请返利机制——一个人开5个小号就能无限刷。你们想过没有？"

**Anvil（evidence-driven）— 同类产品对标**
- 重点：与竞品对比、行业基本功、"这个功能别人怎么做的"
- 风格：丰富的经验参照，用"见过的"和"用过的"做论据
- 参考台词："这个搜索比三年前的淘宝还难用。搜索不是创新题，是基本功。"

**Tradeoff（logical-pragmatic）— 性价比审计**
- 重点：定价、时间投入回报、效率、"值不值"
- 风格：精确计算投入产出比
- 参考台词："完成日常任务需要47分钟，奖励是200金币，而直接买是6块钱。6块钱买47分钟——不值。"

**Trace（evidence-driven/observational）— 细节与文案审查**
- 重点：UI细节、文案措辞、隐藏功能、被忽略的角落
- 风格：安静列出大多数人不会注意到的东西
- 参考台词："设置页第三层有个'高级选项'，点进去后有个功能默认是开着的，但90%用户不知道它在。"

**Gambit（logical-strategic）— 长期价值评估**
- 重点：生态位、护城河、长期留存、竞争策略
- 风格：跳出产品看市场，多步推演
- 参考台词："现在做得不错，但如果大厂3个月内复制这个功能，你靠什么留住用户？"

### 输出格式

```markdown
# [产品名] 八维度体验测评报告

## 测评概要
- 测评日期：
- 产品版本：
- 测评维度：

## 个人测评

### Deduce（逻辑型）评分：X/10
[Deduce的完整反馈]

### Instinct（直觉型）评分：X/10
[Instinct的完整反馈]

...（8个Agent依次）

## 汇总分析

### 共识（多数Agent一致认同的问题）
### 分歧（Agent之间意见不同的点）
### 亮点（多数Agent认可的优势）
### 致命问题（任一Agent发现的严重问题）

## 综合建议
按优先级排列的改进建议
```

## 场景C：决策推演 / 头脑风暴

### 准备阶段
1. 读取全部8份 Profile
2. 明确议题和决策背景
3. 如需要，分配立场（或让Agent自由选择）

### 执行阶段

**Round 1 — 各自表态**
每个Agent基于认知风格给出初始立场和论点。不互相影响。

**Round 2-3 — 自由辩论**
- Agent可以回应其他Agent的论点
- 影响度受 conformity（从众度）和 contrarian（对抗性）参数制约
- 高 rationality Agent会用证据说话
- 高 assertiveness Agent会更积极发言
- 低 conformity Agent不会轻易改变立场

**Final Round — 汇总**
- 统计最终立场分布
- 标记共识点和核心分歧
- 列出每个立场的最强论点

### 输出格式

```markdown
# [议题] 推演报告

## 立场分布
| Agent | 初始立场 | 最终立场 | 是否改变 | 改变原因 |
|-------|---------|---------|---------|---------|

## 核心论点汇总
### 立场A
- [论点及支持者]

### 立场B
- [论点及支持者]

## 共识
## 核心分歧
## 决策建议
```

## 场景D：用户画像模拟

### 准备阶段
1. 读取全部8份 Profile
2. 定义用户画像维度（年龄、职业、技术水平、使用场景等）
3. 为每个Agent分配画像（角色适配层）

### 执行阶段
每个Agent以分配的用户身份 + 自己的认知风格完成指定用户旅程：
1. 首次接触（下载/打开）
2. 新手引导
3. 核心功能使用
4. 进阶探索
5. 留存/流失决策点

### 输出格式

```markdown
# [产品名] 用户画像模拟报告

## 用户旅程地图
| 阶段 | Deduce | Instinct | Throne | ... |
|------|------|------|------|-----|
| 首次接触 | ... | ... | ... | ... |
| 新手引导 | ... | ... | ... | ... |
| ... | ... | ... | ... | ... |

## 流失风险点
## 各画像留存率预测
## 改进建议
```
