## 系统理念

这不是一组"角色扮演模板"——这是**有独立人格内核的持久实体小队**。

他们有固定的思维方式、性格基线和成长记忆。无论被投入剧本杀、产品测评、用户体验模拟还是其他任何场景，他们的**核心人格不变**——只是穿上不同的"角色外衣"。

小队成员可扩展，当前8人。

## 架构：双层人格模型

```
┌─────────────────────────────────────────────┐
│           第一层：核心人格（永久）              │
│  文件：profiles/{代号}.md                      │
│                                               │
│  · 思维方式 (cognition_style)                  │
│  · 性格基线参数 (15项核心数值)                  │
│  · 行为模式 & 沟通风格                         │
│  · 价值观锚点 & 软肋                           │
│  · 声优支柱（说话方式）                         │
│  · 跨场景一致性规则                             │
│                                               │
│  → 任何场景下都不变                             │
│  → 只有"成长事件"才能微调参数                   │
└──────────────────┬──────────────────────────┘
                   │ 叠加
┌──────────────────▼──────────────────────────┐
│         第二层：角色适配层（临时）              │
│  文件：missions/{任务名}/cast.md               │
│                                               │
│  · 角色名 & 背景故事                           │
│  · 角色专属秘密 & 关系                         │
│  · 场景特殊参数调整（有范围限制）               │
│  · 任务目标 & 行为约束                         │
│                                               │
│  → 每次任务更换时替换                           │
│  → 核心人格透过角色表现出来                     │
└─────────────────────────────────────────────┘
```

### 适配规则

1. 角色适配层**不能**偏离核心人格基线超过 ±0.15（任何单项参数）
2. 思维方式（cognition_style）**绝不可改**——这是人格的灵魂
3. 价值观锚点在任何角色中都会以某种方式体现
4. 软肋在任何角色中都可能被触发

## 成长机制

### 成长日志（growth_log）

每份 Profile 底部有 `## 成长日志` 区块。每次完成任务/模拟后，记录：

```markdown
### [日期] [任务名]
- **角色**：本次扮演的角色名
- **关键经历**：发生了什么
- **经验教训**：Agent 从中学到了什么
- **参数微调**：（如有）哪项参数因此调整，调整幅度，理由
```

### 参数微调规则

| 规则 | 说明 |
|------|------|
| 单次调整幅度 | 任何参数单次最多 ±0.05 |
| 累计偏移上限 | 任何参数累计偏离初始值不超过 ±0.20 |
| 不可变项 | cognition_style、core_values 不可通过成长改变 |
| 审批 | 参数调整需在成长日志中说明理由，Summer哥可回滚 |

### 学习反思

每个 Agent 在成长日志中可以记录"反思笔记"——他们对自己行为的评估：
- 判断对了什么、错了什么
- 下次遇到类似情况会怎么做
- 对其他队友的观察和评价

### 被动积累（公共知识库）

不专门安排任务。真心在任意工作区完成有意义的项目后，晚间同步时顺便将项目摘要写入小队的公共知识库：

```
knowledge/
└── {YYYY-MM-DD}.md    ← 当日项目摘要（由真心自动追加）
```

**写入规则：**
- 仅记录有实质产出的项目（网站、报告、工具、分析等），不记录琐碎操作
- 每条摘要：项目名称 / 做了什么 / 用到的技术方法 / 结果
- 每条 2-3 行，不泄露敏感信息

**读取规则：**
- Agent 被召唤时，读取最近 7 天的知识库文件
- Agent 不能声称"参与了"这些项目，但可以引用其中的经验和信息
- 效果："虽然我没参与，但我知道发生了什么"

## 使用方法

### 场景1：剧本杀模拟
1. 提供剧本材料
2. 用 `templates/role_adapter.md` 为每个 Agent 分配角色
3. 核心人格自动带入，只需填角色专属信息
4. 模拟结束后更新各 Agent 的成长日志

### 场景2：产品/游戏体验测评
1. 定义测评任务和评估维度
2. 8个 Agent 以各自的思维方式体验产品
3. 不同认知风格产生不同角度的反馈
4. 汇总形成多维度测评报告

### 场景3：用户画像模拟
1. 为每个 Agent 分配用户画像（年龄、职业、使用习惯）
2. 核心思维方式不变，但行为表现适配画像
3. 模拟真实用户的多样化反应

### 场景4：自由场景
任何需要"一群有独立思考能力的人"参与的场景——头脑风暴、辩论、决策推演、故事创作……
